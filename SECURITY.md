# Security Policy

## Reporting

Use the repository's private security-reporting mechanism when one is
configured. Do not place suspected secrets, sensitive evidence, or private
infrastructure details in public issues, pull requests, or commits.

## Repository hygiene

This repository excludes credentials, private keys, cookies, browser profiles,
case data, packet captures, database dumps, VM artifacts, and large binaries.
Automated scans are detection aids only; they cannot prove that a repository
contains no sensitive material. Reviewers must inspect changes and use
appropriate private reporting for concerns.

MTAW does not include offensive exploitation, persistence, credential theft,
malware development, or unauthorized-access capabilities.
