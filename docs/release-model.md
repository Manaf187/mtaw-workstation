# Release Model

No release exists. Future releases are planned to be versioned, traceable
derived artifacts accompanied by a release manifest and validation evidence.

The repository remains authoritative; a future OVA is only one delivery form.
It must not be treated as the only reproducibility path. A future release must
not include reusable public credentials, secrets, cookies, private
certificates, browser profiles, or case data.

Integrity, authenticity, correctness, and suitability must be assessed and
recorded separately. A checksum alone can detect mismatch against a trusted
reference but does not establish publisher authenticity.

Future release work is expected to distinguish functional reproducibility from
byte-for-byte reproducibility. No such evidence has been created.
